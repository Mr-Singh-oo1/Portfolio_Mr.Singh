This fraud detection pipeline analyzes behavioral patterns, velocity features, and geo-distance anomalies to flag high-risk transactions. It combines domain logic with machine learning to produce precision alerts.

üîç How the Pipeline Works

This pipeline is designed to detect fraudulent transactions using a blend of behavioral analytics, velocity features, and anomaly detection. Here's how it works:

  1. Data Ingestion: Loads raw transaction data from Parquet or CSV format, including timestamps, user IDs, amounts, and device metadata.
  2. Preprocessing: Cleans missing values, converts timestamps, and normalizes numeric fields like amount and geo-coordinates.
  3. Feature Engineering: Adds velocity features (e.g. transactions per hour/day), behavioral metrics (typing speed, navigation speed), and geo-distance calculations.
  4. Fraud Scoring: Applies a trained XGBoost model to assign a fraud probability score to each transaction.
  5. Thresholding: Filters transactions using a top-k threshold (e.g. 1%) to isolate the most suspicious cases.
  6. Alert Generation: Flags high-risk transactions and saves them to fraud-pipeline-alerts.csv for review.
  7. EDA Reporting: Generates a Word report with summary statistics, visualizations, and insights to support audit and analysis.

The pipeline is modular, reproducible, and designed for real-time fraud detection in high-volume environments.
